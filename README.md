**Project Overview**
This project explores a sample sales dataset using SQL. The goal is to analyse product performance, customer behaviour, and regional revenue trends. The dataset was imported into MySQL Workbench, and several analytical queries were written to extract insights.
This proect demonstrates SQL skills including table creation, data importing, aggregation, filtering, grouping, and business-oriented analysis.

**Tools and Technologies**
MySQL Workbench
MySQL Database
CSV dataset
SQL (DDL and DML)

**SQL Tasks perfomed**
1. Create the sales table - A table was created in MySQL Workbench to store the dataset.
2. Import the CSV file - The dataset was imported using the Table Data Import Wizard.
3. Validate the data - Queries were run to confirm that the table structure and imported data were correct.
4. Perform sales analysis - Several analytical questions were answered using SQL, including:
- Which product sells the most?
- Which product generates the highest revenue?
- Which province brings in the most revenue?
- Which province buys the most monitors?
- Which customers made repeat purchases?

**Insights Summary**
- Monitors generated the highest revenue among all products.
- Certain provinces consistently contributed more revenue than others.
- A few customers made repeat purchases, indicating potential loyalty.
- Product performance varies significantly by region.

**What I Learned**
- How to create and structure SQL tables
- How to import CSV data into MySQL Workbench
- How to use GROUP BY, HAVING, and aggregate functions
- How to write clean, readable SQL queries
- How to analyse business questions using SQL
